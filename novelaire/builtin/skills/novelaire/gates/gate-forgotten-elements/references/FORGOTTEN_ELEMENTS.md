# 遗忘元素检查（Checklist + 示例）

本文件用于做“遗忘元素” Gate：在长篇连载/多卷写作中，定期找出已经很久没出现或没推进的关键元素，防止：
- 角色突然消失（读者会问“这个人去哪了？”）
- 支线/线索断掉（读者会觉得“烂尾/弃坑感”）
- 伏笔不回收（读者信任被透支）

> 说明：本文件移植自上游“遗忘元素提醒器”思路，并已按 Novelaire 的目录与工作流做适配（不依赖特定命令或固定数据文件）。

---

## 你要监控什么

### 1) 角色出场频率

目标：找出“应当持续存在但长期没出现”的角色。

建议做法（按需选择一种）：
- 从 `spec/characters/` 读取主要角色名单（主角/核心同伴/核心对手/关键配角）
- 从大纲/细纲里抽取“本卷/本弧活跃角色名单”
- 用户手动给一份“需要追踪的角色清单”

### 2) 情节线 / 线索推进

目标：找出“还没完结但长期没推进”的支线、悬念、任务线、关系线。

元素来源（按需）：
- 大纲/细纲的章节目标（每章“推进了哪个线索/关系/任务”）
- `spec/continuity/` 中的活跃变量/未完成事项（若项目采用）
- 用户手动给一份“活跃情节线清单”

### 3) 伏笔回收

目标：找出“已经埋下但很久没再触碰/提示/回收”的伏笔。

元素来源（按需）：
- 细纲里的伏笔标注
- `spec/glossary/` 或 `spec/objects/` 中的关键物件/术语（如果它们被当作伏笔承诺）
- 用户手动给一份“伏笔清单”

---

## 阈值（默认建议，可按项目调整）

阈值不是硬规则，只是“提醒线”。建议按题材/节奏调参。

- **角色未出现阈值**：10 章
- **情节线未推进阈值**：12 章
- **伏笔未再触碰/未回收阈值**：20 章

你也可以用“卷/弧”单位替代“章数”，只要一致即可。

## 输出口径：Gate Report（示例）

### WARN 示例（可继续但需登记）

```
Target: Forgotten elements
Result: WARN

Findings:
  - severity: WARN
    location: chapters/ (全书)
    problem: 角色“张婶”已 15 章未出现（上次：第 5 章）；如果她仍是活跃配角，读者可能已忘记
    suggested_fix: 在后续 1–3 章安排一次“自然再出现”（电话/消息/目击/旁人提及），或在上游标记她已退场

  - severity: WARN
    location: chapters/ (全书)
    problem: 情节线“寻宝线”已 13 章无进展（上次推进：第 8 章）
    suggested_fix: 在细纲里安排一次推进节点（新线索/阻碍升级/代价暴露），或明确标记为“暂停到第 X 章”

Next step: plan reinsertion
```

### FAIL 示例（影响主线或读者承诺）

当被遗忘元素涉及“读者承诺/主线关键矛盾/核心机制伏笔”时，通常应判为 FAIL：

```
severity: FAIL
problem: 主线关键伏笔“X 的真实身份”在第 3 章明确承诺，但已 25 章未再提示或推进，读者信任风险高
suggested_fix: 立刻在后续章节安排一次可感知的再触碰（提示/反常/代价/线索），并在细纲里规划回收路径
```

## 如何避免误报（很重要）

“没出现”不等于“问题”——需要区分：

### 1) 退场是有意的（retired by design）
- 处理：在上游（大纲/细纲/设定）标记“该角色/线索已完结或退场”，并说明退场条件。

### 2) 暂停是有意的（paused intentionally）
- 处理：标注“暂停到第 X 章/下一卷”，避免每次 Gate 重复报同一条。

### 3) 暗线推进（推进发生在背景里）
- 处理：如果是“背景推进”，也建议偶尔给读者可感知的信号（旁人提及/新闻/后果）。

---

## 修复策略（按成本排序）

### 低成本：一句话的再触碰
- 旁人提及
- 物件/术语再出现
- 情绪或记忆闪回（谨慎）

### 中成本：安排一次小节点推进
- 新线索
- 线索受阻（代价/风险）
- 关系状态发生变化

### 高成本：回修上游
- 发现“原计划需要该角色/线索，但后来写作偏移”
- 处理：先修细纲/大纲，再决定正文怎么补

---

## 最佳实践

- 建议按里程碑节奏跑一次（例如每 5–10 章或每卷结束）
- WARN 允许积累，但必须有“处理计划”；FAIL 优先修
- 如果项目使用了 `@spec:<id>` 约束锚点或其他显式标记，Gate 扫描优先读这些标记（可减少误报）
