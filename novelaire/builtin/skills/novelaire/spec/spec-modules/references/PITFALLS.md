# Modules pitfalls (and fixes)

## 1) Module explosion
- Symptom: lots of modules created “just in case”.
- Fix: enable only when used; keep scope narrow.

## 2) Vague fairness rules
- Symptom: “make it fair” without constraints.
- Fix: define explicit forbidden patterns and reveal prerequisites.

## 3) Duplicating global canon
- Symptom: module entries restate premise/world/system.
- Fix: link to global spec ids; keep module-specific constraints only.

## 4) Modules become outlines
- Symptom: module files encode chapter beats.
- Fix: keep rules reusable; leave beats to outline.
